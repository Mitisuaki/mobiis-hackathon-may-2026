# Plano: Integração API N8N — Listar Conversas e Transcrições

## Contexto
Projeto Angular 21 standalone (MobiisCopilot). Não utiliza `HttpClient` ainda. A rota `/dashboard/meet` atualmente aponta para um `PlaceholderComponent`. A API externa está documentada em `N8N_INTEGRATION.md` (base URL: `http://204.216.160.72:3000`).

## Objetivo
Implementar um service Angular que consuma a API de reuniões/transcrições e um componente de interface que permita listar reuniões, filtrar por assunto, selecionar uma reunião e visualizar suas transcrições. O trabalho será feito em uma branch separada.

---

## Passos

### 1. Criar branch separada
```bash
git checkout -b feature/meetings-transcriptions
```

### 2. Habilitar HttpClient
- Arquivo: `src/app/app.config.ts`
- Adicionar `provideHttpClient()` nos providers da `ApplicationConfig`.

### 3. Criar interfaces de domínio
- Arquivo: `src/app/core/models/meeting.model.ts`
  - `Meeting`: `id`, `subject`, `startDateTime`, `endDateTime`
  - `Transcript`: `id`, `createdDateTime`
  - `TranscriptContent`: `plainText?`, `segments?`

### 4. Criar MeetingService
- Arquivo: `src/app/core/services/meeting.service.ts`
- Métodos:
  - `getMeetings(subject?: string, startDate?: string): Observable<Meeting[]>` → `GET /users/{userId}/meetings` com query params.
  - `getTranscripts(meetingId: string): Observable<Transcript[]>` → `GET /users/{userId}/meetings/{meetingId}/transcripts`.
  - `getTranscriptContent(meetingId: string, transcriptId: string, format: 'plain' | 'segments' | 'full' = 'plain'): Observable<TranscriptContent>` → `GET .../content?format={format}`.
- Constante interna `USER_ID` e `API_BASE_URL` (ou criar `src/environments/environment.ts` se o projeto adotar esse padrão; caso contrário, manter constante no service).

### 5. Criar MeetComponent
- Arquivo: `src/app/domains/dashboard/meet/meet.component.ts`
- Funcionalidades:
  - Campo de busca/filtro por assunto (com debounce).
  - Lista de reuniões retornadas pela API.
  - Ao clicar em uma reunião, carregar e listar suas transcrições.
  - Ao clicar em uma transcrição, exibir o conteúdo (`plainText` ou `segments`).
- Estilo: seguir padrão visual do projeto (classes CSS semelhantes aos demais dashboards).
- Arquivos auxiliares: `meet.component.html`, `meet.component.css`.

### 6. Atualizar rotas
- Arquivo: `src/app/app.routes.ts`
- Substituir `PlaceholderComponent` pela rota `meet` para apontar para `MeetComponent`.

### 7. Adicionar traduções i18n
- Arquivos: `src/app/core/i18n/pt.ts`, `en.ts`, `es.ts`
- Adicionar chaves sob namespace `MEET`:
  - `TITLE`, `SEARCH_PLACEHOLDER`, `NO_MEETINGS`, `TRANSCRIPTS`, `NO_TRANSCRIPTS`, `SELECT_MEETING`, `CONTENT`, `BACK`.

### 8. Ajustar layout/sidebar (se necessário)
- Verificar se o link `Meet` no sidebar continua funcional (já existe, aponta para `/dashboard/meet`).

### 9. Testar e validar
- Rodar `ng serve` e verificar se o componente carrega, lista reuniões, aplica filtros e exibe transcrições.
- Verificar console por erros de CORS ou tipagem.

## Observações
- O projeto utiliza componentes standalone; o novo `MeetComponent` deve ser standalone e importar `CommonModule`, `HttpClientModule` (implícito via `provideHttpClient`), `TranslatePipe`, `FormsModule` e `RouterLink` conforme necessário.
- Não há `environment.ts`; a base URL pode ser mantida como constante no service ou criado `src/environments/environment.ts` — avaliar no momento da implementação.
- A API não requer autenticação.
